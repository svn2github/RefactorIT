Number of Parameters (NP)

This metric counts the number of parameters in a method's signature for a method or constructor. Methods with a large number of parameters are often harder to reuse because they tend to be specialised and this implies that classes are missing from the model. The default limits for NP in RefactorIT are 0 and 4. If the amount of parameters exceeds 4, it is advisable to move an invoked method or pass an object. In general, methods and constructors with more than three parameters are harder to comprehend and to use. Source code with invocations of methods with a long parameter list and invoking methods with similar signatures tends to be error-prone.
