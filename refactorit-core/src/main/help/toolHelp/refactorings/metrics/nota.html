Number of Abstract Types (NOTa)

NOTa counts the number of abstract classes and interfaces. There is no default recommended upper and lower limit in RefactorIT as to how many abstract classes and interfaces there should be for a package. It is common practice that the more stable packages should be abstract packages and equally unstable packages should be concrete packages.
