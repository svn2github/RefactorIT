Executable Statements (EXEC)

This metric counts the number of executable statements, where an executable statement is a statement specifying an explicit action to be taken; i.e. imperative statement. This metric is more objective than the line coverage metrics such as CLOC and LOC because it is not affected by the coding style.

It is generally accepted that as the number of executable statements increases is directly proportional to the complexity of an algorithm, thus a large EXEC becomes harder to maintain and comprehend. For example: 

int a = 13;
int b;
b = a + 4; // <- executable statement
a++, b++; // <- two executable statements
if (b < 6) {
  System.out.println( // <- executable statement
    "Hello, World");
}
