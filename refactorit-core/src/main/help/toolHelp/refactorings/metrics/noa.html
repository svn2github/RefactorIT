Number of Attributes (NOA)

Number of fields declared in class is one measure of its complexity. The more fields a class or interface contains the more problems may appear when extending or implementing it: undesired occasional override of protected or public field can cause malfunctions in program.

Class with large number of fields is probably a candidate for ExtractClass or ExtractSuperClass refactorings.

NOA metric counts number of fields declared in class or interface.
