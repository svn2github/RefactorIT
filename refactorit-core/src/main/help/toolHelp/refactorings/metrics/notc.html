Number of Concrete Types (NOTc)

This metric simply reports the number of concrete (non-abstract) classes. There is no default recommended upper and lower limit in RefactorIT as to how many concrete classes there should be for a package. It is common practice that the more stable packages should be abstract packages and equally unstable packages should be concrete packages.
