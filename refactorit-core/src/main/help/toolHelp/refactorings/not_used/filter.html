Filter Dialog for Not Used

You can narrow down the results of a Not Used query by choosing different settings from the query's filter dialog.

Usage

The filter dialog comes up automatically every time you run a Not Used query. To refine the search, you can also open that dialog later from the results panel by clicking the filter button () after obtaining initial results.



The following table describes the fields in the dialog box:


Name

Description

Interface usages vs. implementation usages

Interface usage means using any method in the subclasses/superclasses hierarchy that has exactly the same signature as the target method (for instance, using overriding methods counts as interface usage).

Implementation usage means using a particular method body. So, overrides don't count, but simply using the method through a subclass does. ( Note that this would also count as interface usage.)

Exclude public and protected members

If checked, RefactorIT will not check public or protected members.
