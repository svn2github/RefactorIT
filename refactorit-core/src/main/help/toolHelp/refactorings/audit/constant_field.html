Staticalizable and finalizable fields

This audit finds fields, which are initialized during declaration to a literal value and are never re-assigned afterwards. It proposes to make such fields 'static final'.

This audit also reports 'static final' fields with lower-case names.


Corrective actions:

Make field 'static final'

Adds 'static final' modifier to field declaration. If You have marked an option 'constant`s name should be uppercase' in audit options, and field`s name is not uppercase, corrective action will also propose to Rename the field.

Example:

before:
    public class A {
        public void int a = 5;
    }
after:
    public class A {
        public static final void int A = 5;
    }

Convert field`s name to uppercase

Action executes Rename refactoring on selected field.

Example:

before:
    public class A {
        public static final int myABCVariable = 5;
    }
after:
    public class A {
        public static final int MY_ABC_VARIABLE = 5;
    }
