Missing blocks

This warning reports missing blocks for 'if', 'while', 'for' and 'do' statements:

Missing blocks can confuse You:

    if(something)
        doSomething1(); // this statement is conditional
        doSomething2(); // this statement is invoked always

This is possible a bug, because maybe You had in mind something like this: 

    if(something){
        doSomething1();
        doSomething2();
    }

To suppress this warning, you can either disable it from the filter dialog or replace the suspicious semicolon with {}.


Corrective actions:

Surround with brackets

Action adds missing {} to statement.

Example:

before:
    if(something)
        doSomething();
after:
    if(something){
        doSomething();
    }
