Audit

Audit warns about suspicious or redundant code such as assigning a variable to itself, local variables that have the same names as fields, and unused import statements. Audit can be implemented on separate files or packages or on entire projects. For some warnings there are automated fixes such as Clean Imports for the Unused imports warning.

Sorting Results

Audit results can be sorted by priority, density of results per file, warning type, and location (default setting). To switch between sort modes, click the appropriate column headers in the results table. Density of results are currently calculated with the following algorithm:

d = 0
for each warning in file
  d = d + priorityCode(warning)
  d = d / lineCount(file)

Every warning has a different priority, for instance warnings about unused imports have a low priority, whereas field and variable self-assignment have a higher priority. The default values are: 10 for low priority, 20 for normal priority, and 50 for high priority. The priorityCode in the above formula is read form the refactorit.cfg file (from properties named audit.violation-priority-score.xxxx).

Corrective actions

Most of the audits have a feature we call "Corrective action". This feature allows to do an automated fix, that will improve the code and make it more clear, proper or faster.

To run corrective actions you need to run Audits and then do right click on an item from the results panel. Corrective actions located in the very top of the appeared popup menu. If you have selected more than one item and those items correspond to the different categories, then you will see those categories. Category may be in bold: it means that there are located applicable corrective actions; otherwise you will found only so called "@skip" action. This action adds @skip tag near the violation code, that will make Audit to skip this place next time.

Audits report generation

Audits results can be saved to files in various formats, so they can be analysed by spreadsheet tools, or, for example, sent via e-mail to the development team as part of the nightly build. To do this - select "Export" on the Results Panel.
