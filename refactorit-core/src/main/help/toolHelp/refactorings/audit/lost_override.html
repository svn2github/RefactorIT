Possible method lost override

If two methods in hierarchy have same names, but different parameters, this is probably a 'lost override'.


    class Fruit {
        public void getName(){
            return "unknown fruit";
        }
    }
    
    class Apple extends Fruite {
        public void getName(boolean b){
            return "apple";
        }
    }
    
    class Test {
        public void test(){
            Fruit fruit = new Apple();
            System.out.println("" + fruit.getName());
        }
    }

Maybe, someone changed signature of method 'getName()' in Apple class, but didn`t change signature of 'getName()' in Fruit and didn`t update method usages. Test.test() method will print 'unknown fruit' to console.
