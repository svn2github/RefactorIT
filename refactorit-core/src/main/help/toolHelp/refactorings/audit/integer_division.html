Integer division in floating point context

This warning reports places of integer division in places, where the result is expected to be floating point type.

Try this code:

int a = 1;
int b = 2;
float c = a/b; // violation
float d = ((float) a)/b; // no violation
System.out.println("c == " + c); // c == 0.0
System.out.println("d == " + d); // c == 0.5

This audit rule analyzes not only assignment context (see example above), but any kind of floating point contexts: method/constructor invocation, arithmetical expressions etc.


Corrective actions:

Add cast to 'float'

Adds cast to 'float' for one part of division expression.

Example:

before:
    float c = a/b;
after:
    float c = ((float) a)/b;
