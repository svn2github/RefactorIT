Shadowed Fields

This warning reports local variables whose names matches the names of fields in the same class, optionally including constructors' and setters' parameters. An automatic fix is available form the pop-up menu, renaming the local variable.

Example 1: 

    class Test {
        private int a;
        
        void m() {
            int a = 5; // violation!
            ...
        }
    }

Corrective actions:

Rename field

Action suggests to rename field. See Rename field refactoring action.
