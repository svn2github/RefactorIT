Where Used

Usage

Where Used is the reverse of Show Dependencies and finds usages of packages, types, members, and constructors inside the source path of a project. Note: Usages in compiled classes (those in the class path) are not reported, yet you can search for usages of compiled classes or their members. Usages are grouped by the location where they occur: by package, type or source, and line number. Additionally, places that cause usage to be reported are highlighted, as shown in the following screen shot:



Double clicking on a reported usage brings you to the exact location where the usage occurs. Additionally, you can right-click a package, type, method, or constructor reported in the Target column and run applicable refactoring operations on it.
