Renaming a Package

Renaming a package changes the name of a package and all references (FQNs, package and import statements) in source files are updated accordingly.

Usage

In the following dialog box, package com will be renamed to org:



Note that with these settings, com.company will not be renamed because "rename prefixes of all matching packages" is turned off. For safety reasons, this is the default behaviour. Although packages are not actually hierarchical in Java, there is an option to treat them as such and rename all packages starting with a particular prefix, as illustrated in the following screen shot:



With the settings from the dialog box above, all packages that have names starting with com will be renamed so they start with org instead. The package com will be renamed to org and the package com.company will be renamed to org.company.
