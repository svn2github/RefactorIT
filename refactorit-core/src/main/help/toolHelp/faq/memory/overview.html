Increasing memory allocation (highly recommended)

Borland JBuilder

You should check bin/jbuilder.config file for such parameters:
     vmparam -Xms32m
     vmparam -Xmx128m

and increase -Xmx like this:
     vmparam -Xms96m
     vmparam -Xmx256m

Since JBuilder2005 parameters are:
     vmmemmin 32m
     vmmemmax 128m

and vmmemmax should be increased like this:
     vmmemmin 32m
     vmmemmax 256m

Oracle JDeveloper

In JDeveloper you can change an option in the jdev/bin/jdev.conf file from e.g.:
     AddVMOption -Xmx256M

to:
     AddVMOption -Xmx512M

SunTM ONE/Java Studio / NetBeans

Memory configuration is saved in the SunTM ONE Studio/NetBeans 3.6 and earlier in file bin/ide.cfg. In NetBeans 4.0 and later it is saved in file etc/netbeans.conf. The default entry should appear as:
       -J-Xverify:none -J-Xms24m -J-Xmx96m

You can modify the value of the -J-Xmx parameter to be 128 Mb via:
        -J-Xverify:none -J-Xms24m -J-Xmx128m

Note: It is strongly advised to set the startup memory (-J-Xms) to the amount that SunTM ONE Studio/NetBeans requires for your system.

Eclipse

On Windows you can create a link to the eclipse.exe and in the Properties of the link set the Target property as following:
C:\Eclipse\eclipse.exe -vmargs -Xms64M -Xmx512M
On Unix you can create a shell script with a single call to eclipse binary or use any other method of passing parameters to the executable.

The stand alone version

Go to the file named refactorit.lax in RefactorIT's home directory and allocate the initial and maximum amount of memory. If necessary, add the following two lines to this file:
        lax.nl.java.option.java.heap.size.initial=128M
        lax.nl.java.option.java.heap.size.max=256M
