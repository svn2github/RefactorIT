How to refactor a small portion of a project

If you are only developing subsection of a much larger project, then you may wish to set the subdirectory as a "source path" for RefactorIT. In order to satisfy dependencies, RefactorIT needs to know the ".class" files of the rest of the project. This is accomplished by adding the directory containing these classes to the "classpath" which may or may not contain your current development classes. Version instructions:

Stand alone version

  1. Go to the Options dialog box (Project->Options) or, create a New Project. (Hint: Use double-click to edit.)
  2. Set the source sub-directory you want to work with as the "sourcepath." This should only be one entry.
  3. Set the directory containing all compiled classes as the "classpath." Add any other required libraries as needed.
  4. Click OK and select File->Save to save the new project settings.
  5. Reload the project via File->Open.

SunTM ONE Studio / NetBeans version

  1. Go to the RefactorIT Options dialog box via Tools->RefactorIT->RefactorIT Options.
  2. Enable "Specify classpath and sourcepath".
  3. Set your source sub-directory you want to work with as the "sourcepath". This should be only one entry.
  4. Set the directory containing all compiled classes as the "classpath." Add any other required libraries as needed.
  5. Restart the IDE to be sure the changes take effect.

JDeveloper9i

  1. Go to the Project Settings.
  2. Set your source sub-directory you want to work with in Common/Input Paths/Java Source Path.
  3. Add the directory containing all compiled classes to "Configurations/Development/Libraries/Selected libraries".
  4. Click OK to save the changes.
