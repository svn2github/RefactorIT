RefactorIT does not appear under Eclipse

You need to run Eclipse with the -clean command line parameter the first time after RefactorIT installation in order to refresh the plugins cache of Eclipse.

Can not perform refactorings on selected item

Usually this means that the class you clicked on/in is out of RefactorIT scope, i.e. it is located outside of the sourcepaths set in RefactorIT project properties. By default the paths are autodetected basing on the IDE project settings, but can be overriden as well. Check the RefactorIT Help for the detailed instructions.

RefactorIT is installed but functionality is not working

Sometimes RefactorIT seems to be installed properly, but none of the functions work. In this case check you IDE logs (location depends on the IDE) for such an error stack trace:
     java.lang.NoClassDefFoundError: rantlr/TokenStream
     at java.lang.ClassLoader.defineClass0(Native Method)

This means that installer failed to install a special version of antlr.jar, which is needed for correct work of RefactorIT. You can get it from: http://www.refactorit.com/files/jarsonly/refactorit.zip It must be located in your IDE in the same directory as refactorit.jar.
