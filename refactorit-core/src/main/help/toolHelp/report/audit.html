Audits

HTML report

This kind of report contains two distinct parts:

The first one is a summary of found violations, grouped by category. High/Normal/Low mean the priority of those violations. The larger priority, the worse it is for your code - it may be potentially a very buggy place. There is also a number near to the each audit name. This number means the amount of found violations. You may click on the audit name and browser will show you the detailed information about this audit and it's violations.

The second one is a detailed information. It also has two parts:

  • Header

    [top] link makes possible go back to the summary. The color of the header means the priority of this audit. Yellow means Low, Blue means normal and red means high.

  • Violations list

    This part contains violation name, it's location in the source code and density. Some violations provide extra information about what shall be done in order to improve the code. It is located in the Corrective Action column.
