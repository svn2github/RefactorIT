Available formats

RefactorIT provide predefined formats that are often in use. Each format has it's own goal, so choose exactly that format what suits your needs the most

HTML format

This format produces a nice report that may be opened by a typical html browser (Internet Explorer, Mozilla, Opera, etc.). Html format is usually used for a source code state fast review. This format does not provide an ability for a machine analysis, but is best for the human reading.

Plain Text format

Plain text format: This format produces a report where results are separated by delimiter '\t' (tab). It is hard to read, but is good for importing into a different data processing tools (Excel, Spreadsheet, number of databases, etc.)

CSV format

This format is similar to the plain text format except it uses comma as a delimiter and escapes double quotes " and comma , in data text. The best variant for importing results into a different data processing tools (Excel, Spreadsheet, number of databases, etc.)

XML format

Almost pure machine format. Although it is readable and writable by human, it is meant for a different purpose. This format provide the most of the results data and may be used to generate other formats you may need.
