Please NOTE: this information applies only to versions older than 4.0. In Netbeans v.4.0 all those settings are detected automatically.

Autodetection of Project Settings

RefactorIT attempts to detect the project source and class path from filesystems mounted in the Filesystems tab of the Explorer window. Thus, RefactorIT should be able to detect your settings properly without manual intervention. Source path and class path are autodetected as follows:

  • Each read-write and visible filesystem marked to be used in the compilation process is added to the source path.

  • The "Read Only" and "Hidden" attributes must be set to False in the Properties tab of the filesystem.

  • The "Use in Compiler" attribute must be set to True in the Capabilities tab of filesystem.



  • Each filesystem that is visible and marked to be used either in the compilation process or for Javadoc comments is added to the class path.

  • The attribute "Hidden" must be set to False in the Properties tab of the filesystem.

  • The "Use in Compiler" or "Use as Documentation" attribute must be set to True in the Capabilities tab of the filesystem's properties.


You can check which source and class paths have been detected via the RefactorIT Options dialog.
