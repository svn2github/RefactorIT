Troubleshooting and Limitations

The VCS support of RefactorIT was tested with CVS only.

Since RefactorIT uses the same VCS commands that are accessible through NetBeans/Sun ONE Studio's user interface, it is assumed that all VCS commands work properly when invoked manually within the Sun ONE Studio/NetBeans, therefore, please, ensure that all the commands of the VCS are functional before running RefactorIT on a VCS file system.


If REMOVE fails for CVS
If your server version is newer that the version of the cvs client Sun ONE Studio/NetBeans was configured to use (including internal cvs-client), then REMOVE invoked on a file sometimes results in "cvs [server aborted]: no such directory '\<name_of_parent_directory\>'". In this case, try to use another cvs client, for example command line client cvs.exe of the WinCvs.


The complete log of all the VCS commands called by RefactorIT together with accompanying information is saved into the ide.log file which is located under the system directory of your NetBeans/Sun ONE Studio user home directory (for example, /home/username/.netbeans/3.5/system/ide.log).
