Cache

RefactorIT maintains an internal cache to minimize the parsing files in a project with each refactoring operation or query. Thus, when you make a change to a single source file, only that file will be parsed.

The result is an improvement in performance, however, sometimes changes fail to update properly and RefactorIT finds deleted classes or reports that a name as in use after the element that has been deleted. Often this occurs when a source or class file is changed and part of the cache is invalidated.

When you notice such behaviour, you have two tools at your disposal to correct it: Clean cache and Rebuild cashe. Both tools can be accessed from the RefactorIT menu:

  1. Rebuild has less impact on performance and tries to rebuild outdated parts of the cache.

  2. Clean forces RefactorIT to entirely purge its cache and the next refactoring or query will build the cache from scratch.

If a Rebuild does not fix the problem, then select Clean. It will take more time to parse all source and class files, but it will fix anomalies experienced due to an outdated cache.
